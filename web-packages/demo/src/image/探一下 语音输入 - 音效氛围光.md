# 语音输入音效氛围光动画规范

## 动画阶段设计

### 第一阶段：监听状态
- **持续时间**：3.4秒
- **纹理运动**：从右向左移动(UV.x偏移0.5)
- **透明度变化**：
  - 0~0.6秒：渐显(透明度0→1)
  - 0.6~2.4秒：完全显示(透明度1)
  - 2.4~3.4秒：渐隐(透明度1→0)
- **音量检测**：
  - 在2.4秒时检测音量：
    - 超过阈值：计划在3.4秒进入第二阶段
    - 未超阈值：立即生成新监听纹理(循环)
- **纹理混合**：
  - 当新旧纹理生命周期重叠时，使用透明度混合

### 第二阶段：音乐输入状态
- **基础持续时间**：2.4秒
- **纹理配置**：
  - 纹理A：
    - 0秒生成，从右向左移动(UV.x偏移1.0)
    - 0~0.4秒：渐显
    - 0.4~1.9秒：完全显示
    - 1.9~2.4秒：渐隐
  - 纹理B：
    - 0.5秒生成，从右向左移动(UV.x偏移1.0)
    - 0.5~1.0秒：渐显
    - 1.0~1.9秒：完全显示
    - 1.9~2.4秒：渐隐
- **音量检测**：
  - 在1.9秒时检测音量：
    - 超过阈值：立即生成新纹理组(A+B)
    - 未超阈值：等待当前纹理组完成渐隐
- **状态转换**：
  - 当没有新纹理生成且所有纹理完成渐隐后，自动返回监听状态

## 实现要点

1. **纹理生命周期管理**：
   - 每个纹理独立维护自己的状态(fadeIn/showing/fadeOut/hidden)
   - 使用时间戳精确控制各阶段转换

2. **多纹理混合**：
   - 同时存在的纹理最多4个(2个渐隐+2个渐显)
   - 使用alpha混合公式：`finalColor = tex1.rgb * tex1.a + tex2.rgb * (1-tex1.a)`

3. **音量触发逻辑**：
   - 使用精确时间检测(±0.05秒容差)
   - 每个纹理只触发一次检测

4. **状态机设计**：
   ```mermaid
   stateDiagram
     [*] --> Listening
     Listening --> Input: 3.4s & volume>threshold
     Input --> Input: 1.9s & volume>threshold
     Input --> Listening: all textures hidden
   ```

## 参数配置

| 参数 | 监听阶段 | 输入阶段 |
|------|---------|---------|
| 总时长 | 3.4s | 2.4s |
| 渐显时间 | 0.6s | 0.4s(A)/0.5s(B) |
| 渐隐开始 | 2.4s | 1.9s |
| 移动距离 | 200px | 400px |
| 检测时间点 | 2.4s | 1.9s |
| 音量阈值 | 0.5 | 0.5 |
